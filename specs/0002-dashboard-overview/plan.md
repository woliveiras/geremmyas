# Plan: Dashboard overview HTML generation

## Approach

Add the `dashboard` command to `cli.go` switch. The command orchestrates:
parse (0001) → render HTML → write README. HTML generation uses Go templates
with embedded assets.

## File structure

```
internal/cli/
├── cli.go                          # add "dashboard" case
├── dashboard.go                    # runDashboard() entry point
└── dashboard/
    ├── parser.go                   # from spec 0001
    ├── types.go                    # from spec 0001
    ├── renderer.go                 # RenderDashboard(data, outputDir)
    ├── renderer_test.go
    ├── readme_gen.go               # GenerateReadme(data) → string
    └── readme_gen_test.go

dashboard_assets/
├── templates/
│   ├── layout.html                 # shared header/footer/nav
│   ├── index.html                  # overview page
│   ├── family.html                 # family detail page
│   └── spec.html                   # spec detail page
└── css/
    └── style.css                   # all styles, dark/light theme
```

## Key decisions

1. **Layout template**: Shared `layout.html` with `{{template "content" .}}`
   blocks. Each page defines its content block. Avoids duplication of nav/head.

2. **Markdown rendering**: Evaluate goldmark vs simple approach. If goldmark is
   too heavy a dependency, start with `<pre>` blocks and add goldmark later.
   Decision: try goldmark first — it's the standard Go markdown library.

3. **README generation**: Separate function that returns a string. Easier to
   test in isolation. Written to `specs/README.md` by the command orchestrator.

4. **sdd template update**: Modify `project/templates/specs/README.md` to add
   a section explaining that `specs/README.md` is auto-generated by
   `geremmyas dashboard` and should not be hand-edited. Keep the conventions
   and frontmatter docs in the template since they're useful for agents creating
   new specs.

## Dependencies

- Spec 0001 (parser + types) — must be complete first
- Possible new dep: `github.com/yuin/goldmark` for markdown rendering
- `go:embed` for dashboard_assets/
